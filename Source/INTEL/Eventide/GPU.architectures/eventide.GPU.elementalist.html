ELEMENTALIST
Xe5

The fifth-generation Xe architecture represents a complete reimagining of Intel's GPU silicon. Wider shader cores. Deeper wave dispatch. Native FP64. A next-generation matrix engine that makes dedicated AI accelerators unnecessary. And a ray tracing unit that handles BVH4 traversal, ReSTIR, and VR stereo rendering in hardware. Built on Intel 04A and connected over FOveros 2.0 photonic fabric — up to four tiles operating as one.

01
SHADER ARCHITECTURE · GENERATION 5
Xe5 Core

The Xe5 shader core is the widest, deepest execution unit Intel has ever built for graphics. Each Xe5-core contains 512 unified shaders — double the 256-shader Xe4E — organised into 32-wide wavefronts with out-of-order wave dispatch. The front-end decodes up to 4 instructions per clock per wavefront, feeding a 512-entry reorder buffer that allows waves to execute non-sequentially based on data readiness. This is a technique borrowed from CPU microarchitecture: waves that hit cache misses yield to waves with data ready, eliminating pipeline bubbles that plague in-order GPU schedulers. The L0 instruction cache per 16-core cluster eliminates instruction fetch bottlenecks for shader-heavy workloads. Native FP64 support — not emulated through FP32 pairs — delivers double-precision compute performance at 1/2 FP32 rate, making the E1080 viable for scientific workstation use without a discrete GPU. Compared to the Xe2 generation in Battlemage, the Xe5 core delivers a 2.2× improvement in instructions retired per clock.

AI MATRIX ACCELERATION · GENERATION 5
XMX5

XMX5 is the most significant generational leap in Intel's matrix acceleration history. Each XMX5 unit delivers 2× the throughput-per-clock of XMX4E at equivalent precision, but the real story is precision expansion. XMX5 introduces native INT2 and FP4 execution — sub-byte precisions that quadruple effective throughput for quantized AI models. For XeSS 5.0 frame generation, the XMX5 units generate entire intermediate frames using a temporal neural network that runs at native resolution, eliminating the artifact-prone upscaling step used by competing solutions. The dedicated 512 KB L1 AI SRAM per XMX5 cluster holds model weights locally, avoiding the memory bandwidth wall that limits matrix throughput on traditional architectures. Sparse activation skipping — where zero-valued matrix elements are detected and bypassed in hardware — delivers up to 1.8× effective throughput on real-world models with typical 40–60% sparsity.

RAY TRACING · GENERATION 5
RTU5

The RTU5 ray tracing unit implements fourth-order bounding volume hierarchy (BVH4) traversal — meaning each BVH node contains four children instead of the industry-standard two. This halves the number of tree levels the hardware must traverse for equivalent scene complexity, delivering a 2× improvement in traversal efficiency before any algorithmic optimisation. Hardware ReSTIR (Reservoir-based Spatio-Temporal Importance Resampling) evaluates hundreds of candidate light paths and selects the most important ones for shading — producing converged global illumination at a fraction of the ray budget required by brute-force path tracing. For VR workloads, the RTU5 introduces stereo BVH caching: the acceleration structure built for the left eye is retained and re-traversed for the right eye with only the viewpoint transform applied, eliminating redundant BVH construction and delivering near-2× VR ray tracing throughput compared to rendering each eye independently.

PROCESS NODE · MULTI-TILE PHOTONIC FABRIC
Intel 04A + FOveros 2.0

The Xe5 GPU tiles are manufactured on Intel 04A — the same world-leading process node used by the Eventide Compute Tile. But what makes the E1080 unprecedented is FOveros 2.0: Intel's second-generation photonic interconnect fabric that binds up to four GPU tiles into a single coherent device. Unlike organic substrate interconnects that measure latency in nanoseconds and bandwidth in hundreds of GB/s, FOveros 2.0 uses on-package silicon photonic waveguides to deliver sub-2-nanosecond tile-to-tile latency at over 3.2 TB/s aggregate bisection bandwidth. The operating system and application see one GPU, one memory space, one command queue. Shader work distribution across tiles, cross-tile L2 cache coherency, and synchronized display output are all handled in hardware by the FOveros tile controller — with zero driver overhead. This is not SLI. This is not CrossFire. This is one GPU, built from four tiles, with single-GPU programming semantics.

MEMORY ARCHITECTURE · KACHE KORE + ZAM
Unified Memory

The Xe5 tiles share a unified memory architecture that bridges on-die cache, on-package bLLC, and system DRAM into a coherent hierarchy. The centerpiece is Kache Kore — a 4 GB bLLC (bandwidthLLC) fabricated on Intel 04A alongside the compute tiles. For the GPU, Kache Kore acts as an enormous L4 cache that absorbs the working set of most 1440p and many 4K game frames entirely on-package, eliminating the latency penalty of DRAM round-trips. Each Xe5 tile maintains its own L2 cache (8 MB/tile, 32 MB total across ×4 E1080), with hardware cache coherency across tiles via FOveros 2.0. When a tile's L2 misses, the request hits Kache Kore at ~8 ns latency — comparable to dedicated GDDR7 on discrete GPUs — before falling through to Intel Z-Angle Memory (ZAM) mounted directly on the FOveros 2.0 photonic substrate. Unlike conventional DDR6/LPDDR6, ZAM is bandwidth and density-optimized for sustained high-throughput workloads, delivering 2–3 TB/s per controller and an effective 5–6 TB/s across both dual-channel ZAM controllers — dwarfing the bandwidth envelope of any discrete GDDR7 stack. Smart Memory Tiling automatically pins frequently-accessed texture mipmap levels and shader constants in Kache Kore, ensuring that bandwidth-critical data stays on-package.
